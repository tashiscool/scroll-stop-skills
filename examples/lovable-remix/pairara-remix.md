# Pairara Remix Prompt Pack

This example shows how to turn the exported file:

`~/Downloads/pairara.com.2026-03-26T14_54_19.235Z.json`

into a complete Lovable landing-page build plan.

## What The Export Says

### Brand Feel
- Light scheme
- Primary accent: `#F2642C`
- Background: `#FAFAFA`
- Primary text: `#17171C`
- Font: `Plus Jakarta Sans`
- Rounded pill buttons
- Modern, calm, trust-first dating product

- navbar

- region checker
- footer

## Lovable Prompt Pack

### 1. Homepage Foundation Prompt

```text
Create a premium responsive landing page for a modern trust-first dating app inspired by this Pairara-style structure. Use a light visual system with a warm off-white background, strong dark text, Plus Jakarta Sans typography, bright orange accent buttons, and fully rounded pill controls. The page should feel thoughtful, modern, and calm rather than flashy.

Build these sections:
- navbar
- hero with headline, supporting copy, and two CTAs
- a profile / compatibility proof card in the hero area
- key features section
- a region-check section
- clean footer

Keep the whole experience polished on desktop and mobile.
```

### 2. Hero Prompt

```text
Create the hero section for this dating app landing page. Keep the layout premium and responsive. Use:
- a small brand label or logo area
- a strong headline about thoughtful matching and compatibility
- one primary CTA button in orange
- one secondary CTA button in a lighter style
- a compact proof card or profile card on the right side

The hero should feel modern, calm, and trust-first, with plenty of whitespace and very clear hierarchy. Keep the button styling fully rounded and the typography in Plus Jakarta Sans.
```

### 3. Navbar Prompt

```text
Add a navbar above the hero. Use the same styles, colors, spacing, typography, and button language already established in the hero. Keep the mobile menu polished and simple. Include the brand on the left and lightweight navigation plus CTA actions on the right.
```

### 4. Features Prompt

```text
Add a key features section below the hero. Keep the same visual system and spacing rhythm. Use a clean 3x2 feature grid with short headings and concise descriptions.

Feature ideas:
- compatibility first
- better filters
- thoughtful profiles
- private and secure
- no ad clutter
- modern web app

Make sure the cards feel clean, slightly elevated, and responsive on mobile.
```

### 5. Region Checker Prompt

```text
Add a region-check section below the features. This section should help people check whether the app is active in their area. Keep the same design language, rounded inputs, clear typography, and premium spacing. Include a state or region selector, a city input, and short supporting copy that explains the waitlist fallback if there are not enough users yet.
```

### 6. Footer Prompt

```text
Add a minimal footer using the same visual language as the page. Include the logo, a short trust-first product statement, and a few lightweight links like Privacy, Terms, and Login. Keep it clean and spacious.
```

### 7. Dating-App Rewrite Prompt

```text
Rewrite the whole page to fit our dating app brand and heart-logo identity. Keep the same Pairara-style structure, but shift the copy so it feels warmer, more premium, and more emotionally resonant. Keep the trust-first dating angle and make the hero, feature blurbs, and CTAs feel more brand-specific.
```

### 8. About Page Prompt

```text
Create an About page following the styles from the homepage. Keep the same typography, pill-button language, orange accent system, rounded cards, and calm trust-first tone. The page should explain the product philosophy, how matching works, and why the app focuses on meaningful connections instead of endless swiping.
```

### 9. Mobile QA Prompt

```text
Audit the entire page for mobile responsiveness. Fix any broken spacing, oversized headlines, awkward wrapping, or crowded card layouts. Keep the visual language identical while making the mobile experience feel finished and premium.
```
