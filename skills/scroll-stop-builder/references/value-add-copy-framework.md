# Value Add Copy Framework

Use this when building any app landing page around motion, video, or a scroll-driven hero.

## Core Rule

Do not let the motion do all the selling.

The copy must explain:
1. what becomes better for the user
2. what the app changes under the hood
3. why the visual metaphor connects to the real product value
4. what action the visitor should take next

## The Copy Spine

### 1. Promise
Lead with the valuable outcome, not just the visual.

Use:
- who the app is for
- what gets better
- what frustration is reduced or removed

### 2. Mechanism
Explain why the app can deliver that outcome.

Name the real lever:
- better ranking
- stronger filters
- clearer memory
- faster generation
- less manual work
- higher quality inputs

### 3. Payoff
Translate the mechanism into practical value:
- less wasted time
- fewer bad fits
- stronger output
- calmer workflows
- more useful conversations
- better decisions

### 4. Proof Or Contrast
Show why the promise is believable:
- before/after contrast
- a short “why this works” section
- visible process steps
- example metrics
- product proof cards

### 5. Objection Handling
Include at least one lower-friction or reassurance step:
- region check
- waitlist
- security statement
- privacy explanation
- demo request

### 6. CTA Ladder
Use:
- one primary CTA
- one secondary CTA for visitors who need more confidence first

## Section Rules For Scroll-Stop Pages

### Hero
- promise first
- mechanism second
- 1 primary CTA
- 1 supporting CTA
- no more than 3 short proof bullets

### Sticky Sequence
- each beat should add new understanding
- tie motion beats to value beats
- do not repeat the hero headline three times in different words

### Feature / Benefit Section
- translate features into outcomes
- each card should answer “why does this matter?”

### Proof / Comparison
- show what improves after the app does its work
- contrast generic alternatives with the app’s stronger starting conditions

### Final CTA
- restate the value in simpler language
- reduce friction
- make the next step obvious

## Anti-Patterns

Avoid:
- pure aesthetic copy with no product value
- vague “powerful AI” claims
- generic feature names with no user consequence
- too many equal CTAs
- scroll sections that look beautiful but fail to explain the product

## Quick Test

Before finishing, ask:
- Does the page explain the app’s value add in one sentence?
- Does the motion support the product story instead of distracting from it?
- Does each section add something new?
- Is there one clear primary action and one lower-friction supporting action?
